San Francisco City Streaming

Mobile / GPUI PRO / point cloud optimization / cell-based streaming

A production pipeline for rendering a large city scene based on a point cloud / instancing dataset. I converted a heavy monolithic dataset into a cell-based streaming system with runtime loading, GPUI batch upload and controlled unloading of inactive cells.

1.5 GB → 300 MB source dataset reduction
~9 → 60 FPS runtime performance gain
-90% CPU/GPU load
  • Architecture: designed the city streaming runtime around cells, runtime grid lookup, desired-cell planning, state tracking and controlled unload flow.
  • Data pipeline: implemented conversion from CSV source data to binary chunks, per-cell metadata, runtime catalog and GPUI-ready upload batches.
  • Runtime streaming: loaded only the required city areas around the player and unloaded inactive cells instead of keeping the full city dataset active.
  • Optimization: reduced runtime data volume, batching overhead and rendering cost, eliminating overheating on a mobile device.

Stack: Unity, Mobile, Addressables, GPUI PRO, binary data pipeline, runtime streaming, batching, object pooling.
